# Metadata Extraction Script (`extract_metadata.py`)

This script parses 6502 assembly files (typically generated by `da65`) to extract symbol names, block comments, and inline comments, merging them into the `zelda2.json` database.

## Core Logic

### 1. File Scanning
- Scans the `junk/` directory for all `*.asm` files.
- Processes each file line-by-line using a state-based approach to capture context (pending labels and block comments).

### 2. Detection Patterns (Regex)
- **Segments:** Detects `.segment "PRG(\d+)"` to map data to the correct bank ID.
- **Labels:** Captures line-starting labels (`LabelName:`) while ignoring auto-generated patterns (e.g., `L8000`).
- **Address Markers:** Identifies the first comment block containing the CPU address (format: `; 0xOFFSET $ADDR HEX...`).
- **Borders:** Filters out decorative separator lines (e.g., `; --------`).
- **Useless Comments:** Discards register-only assignment comments like `A = FF` or `X -> 00`.

### 3. Cleaning Rules
- **Semicolons:** Aggressively strips leading and trailing semicolons from all captured text.
- **Whitespace:** Trims all captured strings.
- **Multi-line Blocks:** Concatenates sequential comment lines into a single `block_comment`.

### 4. Merging & Persistence
- **Bank Auto-Creation:** Automatically initializes bank structures in the JSON if they exist in the ASM but not in the DB.
- **Conflict Resolution:** If multiple files provide metadata for the same address, symbols and inline comments are overwritten by the latest encounter, while block comments are concatenated.
- **Mapped At:** Defaults new banks to `$8000` (Banks 0-6) or `$C000` (Bank 7).

## Data Flow
`ASM Files` -> `Regex Extraction` -> `BTreeMap (In-memory Merge)` -> `zelda2.json`

## Future Improvements
- Add support for extracting global equates (currently ignored).
- Implement more sophisticated "mapped_at" detection based on the first encountered address in a segment.
- Support for different ASM syntax variants (e.g. `ca65` vs `asm6` style comments).
